# Meeting Intelligence App — Обзор приложения

**Дата:** 17.05.2026  
**Версия:** актуальная (на основе кода)

---

## О приложении

Веб-приложение для управленческого консультанта. Автоматизирует замкнутый цикл работы со встречами one-to-one: **выход каждой встречи автоматически становится входом следующей.** Убирает ~30% ручной работы — больше не нужно каждый раз вспоминать и копировать контекст из предыдущих разговоров.

**Пользователи:**
- **Даниил** (`consultant`) — читает карточки, комментирует, настраивает AI, отслеживает прогресс
- **Саша** (`assistant`) — вносит данные: создаёт встречи, загружает расшифровки, заполняет анализ

**Стек:** Next.js 16, React 19, TypeScript, Tailwind CSS, Supabase (PostgreSQL + Auth + Storage), OpenRouter API, Vercel

---

## Цикл работы

```
Контекст из предыдущих встреч
        ↓
Сценарий встречи  ←── AI (GPT-4o, Claude и др.)
        ↓
Промпт для расшифровки  ←── AI (на основе сценария)
        ↓
Встреча → запись в Plaud
        ↓
Загрузка расшифровки (.txt / .docx)
        ↓
AI-анализ → протокол (ключевые факты, проблемы, выводы, план действий)
        ↓
Обновление Трека развития руководителя
        ↓
Контекст следующей встречи (цикл повторяется)
```

## Разделы приложения

### 1. Компании `/companies`

Список всех компаний.

**На карточке компании:**
- Название, статус (активна / завершена)
- Количество активных руководителей
- Дата последней встречи
- Прогресс-бар по закрытым встречам

**Права:** создание компании — только `assistant`.

---

### 2. Руководители `/companies/[id]/managers`

Список руководителей выбранной компании.

**На карточке руководителя:**
- Имя, должность, роль в команде
- Тип работы (one-to-one / диагностика)
- Количество встреч, дата последней
- Статус: в процессе / завершён

**Права:** создание и массовое создание (`bulk-create`) — только `assistant`.

---

### 3. Карточка руководителя `/companies/[id]/managers/[managerId]`

Центральный экран. Разделён на **профиль** (слева) и **7 вкладок** (справа).

#### Профиль (статичный блок)

| Поле | Редактирует |
|------|-------------|
| Имя, должность, роль в команде | `assistant` |
| Контекст (кто такой, особенности) | `assistant` |
| Запрос от директора | `assistant` |
| Сильные стороны | `assistant` |
| Слабые стороны | `assistant` |
| Комментарии консультанта | только `consultant` |
| `ai_rules` — кастомные правила для AI | `consultant` |

---

#### Вкладка «Встречи»

Хронологический список встреч (accordion, последняя раскрыта). Создание новой встречи с автозаполнением контекста. Полная карточка каждой встречи (см. раздел «Карточка встречи»).

---

#### Вкладка «Динамика»

Межсессионная динамика по всем встречам руководителя.

**Блоки:**
- **Статус установок** — таблица: встреча / тезис / статус (принята интеллектуально / применяется в действии / стала нормой / не принята)
- **Динамика паттернов** — таблица: паттерн / механика / было→стало / статус (усилился / ослаб / контрпример / без изменений)
- **Договорённости** — разбивка по статусам: обещано / выполнено / перенесено / игнорировано; счётчики и детальный список

Кнопка **«Проанализировать все встречи (ИИ)»** — делает AI-снимок динамики по всем встречам и сохраняет в поле `manager.dynamics_snapshot`.

---

#### Вкладка «Трек»

Живой markdown-документ «Навигационный трек развития руководителя».

**Структура трека (13 секций):**

| ID секции | Содержание |
|-----------|-----------|
| `header` | Метаданные: ФИО, должность, компания, консультант, дата, версия |
| `profile` | Роль, контекст, зона ответственности, стейкхолдеры |
| `chronology` | Хронология сессий — блок на каждую встречу с датой |
| `patterns` | Паттерны мышления и поведения (3.1 устойчивые / 3.2 в изменении / 3.3 слепые зоны) |
| `beliefs` | Установки (4.1 текущие / 4.2 целевые) |
| `progress` | Сдвиги и прогресс (шкалы 0→5 по ключевым осям) |
| `people` | Ключевые люди в орбите и дельта по каждому |
| `tools` | Инструменты и фреймворки (применённые и планируемые) |
| `risks` | Риски и красные флаги |
| `priorities` | Приоритеты на ближайший период |
| `metrics` | 10.1 поведенческие / 10.2 результатные / 10.3 маркеры регресса |
| `open_questions` | 11.1 открытые вопросы / 11.2 гипотезы для проверки |
| `footer` | Пометка «живой документ» |

**Действия:**
- Ручное редактирование текста
- **«Синхронизировать из встречи»** — AI добавляет дельты из последней закрытой встречи (режим `append`)
- **«Синтезировать из всех встреч»** — AI пересобирает весь трек из архива (режим `replace`)
- Показ diff с предыдущей версией (версионирование)
- Экспорт в PDF

---

#### Вкладка «Библиотека»

Документы руководителя (тип: track / roadmap / chronology / other). Используются как контекст AI-агента в чате.

---

#### Вкладка «Чат с AI»

Персональный AI-агент в контексте конкретного руководителя. История диалога сохраняется в БД.

**Системный промпт агента собирается из 6 слоёв:**
1. Методология консультанта (активные документы глобальной библиотеки)
2. Базовая роль + имя руководителя
3. Полный профиль руководителя
4. Кастомные `ai_rules` (если заданы)
5. Трек развития (полный markdown)
6. Все документы компании (хронология, роадмап и др.)

---

#### Вкладка «События»

Промежуточные события между встречами. Источник: приложение или Telegram-бот. Добавление и удаление событий.

---

#### Вкладка «Цели»

Дерево целей и подцелей (до 3 уровней вложенности).

- Статусы: планируется / в работе / выполнено (переключаются кликом на точку)
- Прогресс-бар по каждой цели (слайдер 0–100%)
- Общий прогресс по всем целям
- Редактирование двойным кликом

---

### 4. Карточка встречи

Центральная сущность системы.

**Поля карточки:**

| Поле | Заполняет | Примечание |
|------|-----------|-----------|
| Дата, тип, номер | `assistant` | номер автоинкремент |
| Контекст из предыдущей встречи | Автоматически | при создании встречи |
| Сценарий | `assistant` + AI | генерация по кнопке |
| Промпт для расшифровки | `assistant` + AI | на основе сценария |
| Расшифровка (текст) | `assistant` | загрузка .txt / .docx |
| Ключевые факты | `assistant` + AI | |
| Проблемы и сигналы | `assistant` + AI | |
| Выводы | `assistant` + AI | |
| Сильные / слабые стороны | `assistant` + AI | |
| План действий | `assistant` + AI | |
| Сценарий следующей встречи | AI | на основе анализа |
| `diagnostic_extension` | AI | JSON: установки, паттерны, гипотезы, договорённости |

**Статусная машина (только вперёд):**

```
подготовка → проведена → обработана → закрыта
```

Переход в `закрыта` требует заполненных полей «Выводы» и «План действий».

**Автоматический контекст:** при создании новой встречи поле «Контекст» заполняется из последней **закрытой** встречи с этим руководителем (выводы + план действий + проблемы + комментарии Даниила).

---

### 5. Настройки `/settings`

- Выбор AI-модели (список актуальных моделей с OpenRouter)
- Telegram Chat ID для отправки уведомлений
- **Библиотека консультанта** (глобальные документы методологии)

---

### 6. Библиотека консультанта

Глобальные документы (методология, правила работы, фреймворки), которые включаются в контекст AI для **всех** руководителей и компаний.

- Тип документа: methodology / rules / framework / other
- Тогл вкл/выкл из контекста AI без удаления
- Загрузка из .txt / .md / .docx

---

### 7. Telegram-интеграция

- Отправка сценария встречи как PDF в Telegram
- Уведомления после сессии (резюме + список договорённостей с дедлайнами)
- Приём промежуточных событий через Telegram-бота (они появляются на вкладке «События»)

## Связи между разделами

```
Библиотека консультанта (глобальная методология)
            │
            ↓ системный промпт AI
Компания
  └── Руководитель
        │
        ├── Профиль → используется всеми AI-промптами
        │
        ├── Встречи (история)
        │     ├── Контекст ─────────────────────────────→ Следующая встреча
        │     ├── Расшифровка → AI-анализ → diagnostic_extension
        │     └── Закрытая встреча → Синхронизация Трека
        │
        ├── Трек развития ─────────────────────────────→ Чат с AI (контекст)
        │
        ├── Библиотека документов ─────────────────────→ Чат с AI (контекст)
        │
        ├── Динамика (снимок из всех встреч)
        │
        ├── События (межсессионные заметки + Telegram)
        │
        └── Цели (дерево, статусы, прогресс)
```

## Схема построения промптов

### Промпт 1: Генерация сценария встречи

**Endpoint:** `POST /api/ai/generate-scenario`

```
System:
  Структура сценария — 4 обязательных блока:
    1. ЧЕК-ИН (5 мин) — точный вопрос про состояние прямо сейчас
    2. ПРОВЕРКА ДОГОВОРЁННОСТЕЙ (10 мин) — что было обещано, что нужно проверить
       (не "почему не сделал", а "что случилось в момент, когда должна была начать")
    3. ОСНОВНОЙ БЛОК (25–30 мин) — 1–2 сюжета
       (кейс → диагностический вопрос → анализ → следующий шаг → фиксация итога)
    4. ФИКСАЦИЯ (5 мин) — ≤3 договорённостей, у каждой:
       Дедлайн: [конкретная дата]
       Формат отчёта: [как отчитаться]

  Дополнительно для встречи №3 и далее:
    - В блоке 3 сначала «Приоритет сюжетов» — ровно 2 сюжета с обоснованием
    - У каждой договорённости обязательно Дедлайн и Формат отчёта отдельными строками

User:
  [Контекст предыдущих встреч или "это первая встреча"]
  Номер предстоящей встречи: N
  Профиль руководителя: должность, контекст, запрос директора,
    сильные стороны, слабые стороны, заметки консультанта
  → "Подготовь сценарий"
```

### Промпт 2: Анализ расшифровки

**Endpoint:** `POST /api/ai/analyze-transcription`

```
System:
  Верни ТОЛЬКО валидный JSON без markdown-блоков:
  {
    "key_facts": "...",           // 3–7 пунктов списком
    "problems_signals": "...",    // проблемы, риски, сигналы
    "conclusions": "...",         // главные выводы
    "strengths": "...",           // сильные стороны этой встречи
    "weaknesses": "...",          // зоны роста этой встречи
    "action_plan": "...",         // кто, что, к когда
    "next_scenario": "...",       // темы и вопросы для следующей встречи
    "diagnostic_extension": {
      "installations": [{id, thesis, notes, follow_up_status}],
      "behavior_patterns": [{id, name, mechanics, status}],
      "regression_markers": ["..."],     // 1–3 красных флага
      "verification_hypotheses": ["..."], // 2–3 проверяемых гипотезы
      "commitments": [{text, due, status}]
    }
  }

User:
  Профиль: имя, должность, контекст
  Контекст предыдущих встреч: [если есть]
  Расшифровка встречи: [текст]
```

### Промпт 3: Синхронизация трека из встречи

**Endpoint:** `POST /api/managers/[id]/track/sync-from-meeting`

```
System:
  Верни строго валидный JSON:
  { "updates": [{ "sectionId": "<id>", "mode": "append"|"replace", "markdown": "..." }] }

  sectionId только из:
    header, profile, chronology, patterns, beliefs, progress,
    people, tools, risks, priorities, metrics, open_questions, footer

  Правила:
  - append — для chronology, patterns, people, risks, priorities, metrics, open_questions, tools
  - replace — для header и замены заглушек целиком
  - Для chronology: append блок с датой встречи, 5–12 предложений
  - Не перезаписывай существующее, только добавляй новое
  - Если данных нет — секцию не включай

User:
  Руководитель: [имя], встреча №N, дата: [дата]
  Расшифровка, ключевые факты, проблемы, выводы,
    сильные стороны, зоны роста, план действий
  Фрагмент текущего трека (обрезан до лимита)
```

### Промпт 4: Синтез трека из всего архива

**Endpoint:** `POST /api/managers/[id]/track/synthesize-from-meetings`

```
System:
  Составь ПОЛНОЕ наполнение трека уровня внутреннего консультантского документа.
  Везде mode "replace". Требования по секциям:
  - header: метаданные (ФИО, роль, компания, консультант, дата, версия)
  - profile: роль, контекст, стейкхолдеры
  - chronology: подзаголовок с датой/номером на каждую встречу из архива
  - patterns: ### 3.1 / 3.2 / 3.3 с маркированными списками
  - beliefs: ### 4.1 текущие / 4.2 целевые
  - progress: шкалы 0→5 с обоснованием из фактов
  - people: ключевые фигуры и дельта
  - risks: с уровнем серьёзности
  - priorities: конкретные приоритеты из планов
  - metrics: ### 10.1 / 10.2 / 10.3
  - open_questions: ### 11.1 / 11.2

User:
  Руководитель: [имя], должность, компания
  Профиль и карточка из системы
  Архив всех встреч (все поля по каждой)
```

### Промпт 5: Анализ динамики

**Endpoint:** `POST /api/managers/[id]/dynamics/analyze`

Анализирует поле `diagnostic_extension` всех закрытых встреч:
- Установки и эволюция их статуса по встречам
- Паттерны поведения и их динамика (было/стало)
- Договорённости и их судьба (promised / completed / postponed / ignored)

Возвращает `DynamicsSnapshot`, который сохраняется в `manager.dynamics_snapshot`.

## Сильные стороны

- **Замкнутый цикл реализован** — контекст подтягивается автоматически, нет ручного копирования
- **Трек развития** — серьёзный артефакт с 13 секциями, который AI обновляет секционально (append/replace), не перезаписывая лишнее
- **Многослойный контекст агента** — методология → профиль → ai_rules → трек → документы; агент действительно знает историю
- **Диагностика глубже заметок** — установки, паттерны, маркеры регресса, гипотезы для следующей встречи хранятся как структурированный JSON и визуализируются отдельно
- **Промпты методологически детализированы** — 4 блока сценария, правила для встречи №3+, обязательные дедлайн и формат отчёта
- **Telegram** — отправка PDF, получение межсессионных событий
- **Библиотека консультанта** — глобальная методология подключается к AI одним тоглом
- Тёмная тема, адаптивная вёрстка, toast-уведомления, оптимистичные обновления UI

## Слабые стороны

- **Нет интеграции с Plaud** — расшифровки загружаются вручную
- **Нет поиска** — при росте числа встреч навигация только через accordion
- **Цели оторваны от встреч** — нельзя связать договорённость из встречи с целью напрямую
- **Динамика пустая без AI-анализа** — если расшифровки нет, `diagnostic_extension` не заполняется
- **Одна модель для всех задач** — нет возможности выбрать разные модели для сценария, анализа и чата
- **Нет уведомлений/напоминаний** о предстоящих встречах
