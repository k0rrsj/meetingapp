## Фаза 3: Замкнутый цикл

**Оценка:** 3 дня  
**Документы:** SERVICE_SPEC.md §1, §2

### Задачи

**Логика контекста:**
- [ ] Функция `buildPreviousContext(managerId)` — собирает контекст из последней встречи
- [ ] Интеграция в `POST /api/managers/:id/meetings` — автозаполнение `previous_context`
- [ ] Предупреждение в UI при `context_from_unclosed = true`

**Жизненный цикл встречи:**
- [ ] `PATCH /api/meetings/:id/status` — полная бизнес-валидация переходов
- [ ] Валидация обязательных полей при переходе в `closed` (выводы + план действий)
- [ ] Блокировка полей по статусу в UI (см. SERVICE_SPEC.md §2.3)
- [ ] `MeetingStatusBar` — визуальная полоска статусов с кнопкой перехода

**UI встречи:**
- [ ] Автосохранение текстовых полей (debounce 1500ms)
- [ ] Индикатор состояния сохранения
- [ ] Блок «Контекст» — разворачиваемый, только чтение

**Визуализация прогресса:**
- [ ] Хронологический список встреч (последняя раскрыта)
- [ ] Агрегаты на карточках компаний (счётчики, дата, прогресс-бар)

### Критерий готовности

При создании новой встречи контекст предыдущей подтягивается автоматически. Статус можно изменить только в правильной последовательности. Переход в `closed` без выводов — заблокирован.

## Фаза 5: AI-интеграция (OpenRouter)

**Оценка:** 3 дня  
**Документы:** SERVICE_SPEC.md §3

### Задачи

**Backend:**
- [ ] Создать `lib/openrouter/client.ts` — базовый клиент для OpenRouter API
- [ ] Создать системные промпты в `lib/prompts/`
- [ ] `POST /api/ai/generate-scenario` — генерация сценария
- [ ] `POST /api/ai/generate-prompt` — генерация промпта расшифровки
- [ ] `GET /api/ai/models` — список моделей
- [ ] `PATCH /api/ai/settings` — сохранение выбранной модели
- [ ] Полная обработка ошибок OpenRouter (timeout, 401, 429, 500)
- [ ] Автогенерация сценария при создании встречи (если настроен ключ)

**UI:**
- [ ] Компонент `AiGenerateButton` — кнопка с состоянием (loading / error / success)
- [ ] Страница `/settings` — выбор модели
- [ ] Уведомления об ошибках AI в toast/alert

### Критерий готовности

Нажатие «Сгенерировать сценарий» отправляет запрос, показывает индикатор загрузки, вставляет результат. Модель переключается в настройках. Ошибки API обрабатываются корректно.

**Деплой:**
- [ ] Настроить переменные окружения в Vercel Dashboard
