# Meeting Intelligence App — Полный контекст проекта

## О проекте

Веб-приложение для управленческого консультанта Даниила. Автоматизирует цикличный процесс работы со встречами — сценарий → встреча → расшифровка → анализ → следующий шаг. Сейчас всё делается вручную через Word, Excel, чаты с ИИ и интерфейс Plaud. Задача — собрать в одно место и убрать ~30% ручной работы.

---

## Пользователи

**Даниил** — управленческий консультант. Основной пользователь. Читает карточки, добавляет комментарии, отслеживает прогресс. Заходит перед встречей (подготовиться) и после (просмотреть результат).

**Саша** — ассистент. Вносит данные: сценарии, расшифровки, анализы. Готовит карточки к каждой встрече.

---

## Ключевая функция — Замкнутый цикл

Это главное отличие приложения от простого хранилища документов.

**Выход каждой встречи автоматически становится входом следующей.**

- Анализ, план действий и комментарии Даниила → автоматически подтягиваются как контекст при подготовке следующей встречи с тем же руководителем
- Саша открывает новую карточку — контекст из предыдущей уже внутри
- Даниил видит не просто одну встречу, а цепочку встреч с динамикой

Именно это убирает основную ручную работу — больше не нужно каждый раз вспоминать и копировать контекст из предыдущих разговоров.

---

## Цикл работы Даниила

```
Контекст из предыдущих встреч
        ↓
Сценарий встречи  ←── ветка ИИ под каждого руководителя
        ↓
Промпт для расшифровки  ←── там же, на основе сценария
        ↓
Встреча → запись в Plaud
        ↓
Расшифровка через промпт → протокол
        ↓
Анализ + комментарии Даниила
        ↓
План действий + сценарий следующей встречи
        ↓
(цикл повторяется)
```

---

## Типы встреч

**Диагностика** (разовая, стартовая)
Что происходит в компании, как выглядит функция, какие сложности, что хотелось бы поменять.

**One-to-one / менторинг** (повторяющийся ритм)
Как устроена функция внутри, кросс-функциональные коммуникации, отношения с руководством, приоритеты и задачи, принцип делегирования. Сценарий формируется на основе контекста всех предыдущих встреч.

**MVP — только one-to-one.**

## Структура приложения — три уровня

### Экран 1 — Компании

Список всех компаний. На карточке компании:
- Название
- Количество активных руководителей
- Дата последней встречи
- Общий прогресс по всем руководителям компании
- Статус: активна / завершена

### Экран 2 — Руководители компании

Список руководителей выбранной компании. На карточке:
- Имя, должность, роль
- Тип работы: диагностика / менторинг
- Количество встреч
- Дата последней встречи
- Краткий статус: в процессе / завершён

Основная информация (должность, роль, контекст) — заполняется один раз.

### Экран 3 — Карточка руководителя

**Профиль (статичный)**
- Имя, должность, роль в команде
- Контекст — кто такой, особенности
- Запрос от директора — что сказали про этого человека
- Сильные стороны
- Слабые стороны
- Комментарии Даниила (редактируемое поле)

**История встреч (динамичный)**
Хронологический список встреч. Прогресс между встречами виден визуально. Каждая встреча — раскрываемая карточка.

---

## Карточка встречи — главная сущность

| Поле | Кто заполняет |
|------|--------------|
| Дата, тип встречи, номер | Саша |
| Контекст из предыдущих встреч | Автоматически |
| Сценарий встречи | Саша |
| Промпт для расшифровки | Саша |
| Расшифровка (текст из Plaud) | Саша |
| Ключевые факты | Саша |
| Проблемы и сигналы | Саша |
| Выводы | Саша |
| Сильные / слабые стороны | Plaud / Даниил вручную |
| Комментарии Даниила | Даниил |
| План действий | Саша + Даниил |
| Сценарий следующей встречи | Автоматически на основе анализа |

**Статус карточки:** подготовка → проведена → обработана → закрыта

---

## Комментарии

- Доступны на двух уровнях: в профиле руководителя и внутри каждой карточки встречи
- Видны обоим пользователям
- Учитываются при подготовке следующей встречи

---

## Объём данных

- 3 компании
- 5 руководителей в каждой (~15 человек итого)
- Встречи раз в 1–2 недели с каждым
- ~15 документов на руководителя в начале, далее растёт

## Ключевые требования к решению

- Компактное и ясное — без лишних функций
- Замкнутый цикл — выход встречи автоматически становится входом следующей
- Оба сценария равноценны: подготовка к встрече и просмотр результатов после
- Комментарии Даниила видны всем и учитываются в работе
- Экономит Даниилу минимум 30% рабочего времени
- Удобно для итеративного улучшения и развития
- Доступ по ссылке для обоих пользователей
